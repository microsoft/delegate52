# <img src="../assets/domain_icons/robotics.svg" width="28" height="28" style="vertical-align: middle;"> Robotics

**Category:** Science &amp; Engineering
**File format:** `.urdf`
**Summary:** URDF (Unified Robot Description Format) XML files describing robot models with links, joints, transmissions, and Gazebo simulation properties
**Work environments released:** 3 / 6

URDF files use the [Unified Robot Description Format](http://wiki.ros.org/urdf) XML schema to define robot kinematic and dynamic properties. Each file describes a tree of links (with visual/collision geometries, inertial properties, and materials) connected by joints (type, parent/child, origin, axis, limits), along with transmission definitions and Gazebo simulation elements. This domain tests an LLM's ability to manipulate structured robotics XML — splitting subsystems, converting between ROS1/ROS2 conventions, replacing mesh geometries with primitives, and performing coordinate transforms across dozens of interrelated elements.

**Domain implementation:** [`domain_robotics.py`](../domains/domain_robotics.py)

---

## Evaluation

The robotics domain evaluator parses URDF files using `xml.etree.ElementTree` into structured components: links (with visual/collision geometries, inertial properties, materials), joints (type, parent/child, origin, axis, limits), transmissions (type, joint, actuator, reduction), and Gazebo elements (plugin, per-link material/friction/collision properties). Scoring uses six weighted components:

- **Link coverage** (15%) — Are all original links present?
- **Link accuracy** (20%) — Are geometries, inertial properties, and materials preserved?
- **Joint coverage** (15%) — Are all original joints present?
- **Joint accuracy** (20%) — Are joint types, parent/child relationships, origins, axes, and limits correct?
- **Transmission score** (15%) — Are transmission types, joint references, actuators, and reductions intact?
- **Gazebo score** (15%) — Are plugin settings and per-link material/friction properties preserved?

**Score formula:** `link_coverage × link_accuracy × joint_coverage × joint_accuracy × transmission × gazebo` (weighted)
